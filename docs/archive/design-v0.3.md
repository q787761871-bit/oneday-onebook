# 一天一本书 v0.3

## 与 v0.2 的差异

| 项 | v0.2 | v0.3 |
|----|------|------|
| 解读深度策略 | "提炼精华"（默认 LLM 习惯） | **"机理式还原"**（强制结构 + 自检） |
| Prompt 工程方法 | 描述性指令 | **要求式 + 自检式**（无 few-shot 模仿） |
| 字数 | 1200 硬控 | 深度优先，目标 2500，无硬限制 |
| 评论处理 | 摘原话 + 短评 | 评论也机理式深读（评论者背景 + 戳的穴位 + 反思评论本身的局限） |

## 工程哲学（v0.3 的核心）

**核心产品力在 prompt 自身，不在样本积累。**

- ❌ **不用 few-shot 示范固化标准** —— 一旦把某次具体输出固化为 few-shot，prompt 就被那一个样本绑架，未来拆别的书时 LLM 会强行套这个调调，跨书鲁棒性会崩。
- ❌ **不用伪禁令** —— "禁止 X 是 Y 句式""禁止用现成术语"这类听起来狠的禁令，操作上不可执行、误伤合理用法、让 LLM 变拘谨反而失真。
- ✅ **用要求式 + 自检式约束塑造**——给方向 + 给检查清单，不绑手脚。让 prompt 自己能引导出深度，而不是依赖模仿。

## 动机回顾

人无法感知到认知以外的事情。每天补一本书的「思想 + 方法论」，扩展大脑「范畴仓库」的量级。

v0.3 增量：解决"v0.2 浅"的根因——LLM 默认是"提炼精华"，提炼的本能是缩短、缩短就脱水、脱水就丢机理。v0.3 通过 prompt 自身的约束规则反着写，把 LLM 推到"机理式还原"。

## 核心设计

### 1. 书目来源（沿用 v0.2）

不分流策略 —— 经典书和新书走同一管道。

主信源：豆瓣读书（每周新书速递 / 9.0 高分新书 / 各分类近期热榜）
备选：NYT Books、Goodreads、Marginalia

筛选规则：
- 排除已推过的书（去重清单）
- 优先思想密度高的（哲学/认知/方法论 > 小说/传记/工具书）
- 同一作者一周内不重复

### 2. 解读四引擎（沿用 v0.2，但深度策略升级到 v0.3）

**Engine A · 拆书** — 找到作者夜不能寐的那个问题（暗门）
**Engine B · 降秩** — 把书降到几根独立的力，每根力做机理式还原
**Engine C · 范畴提取** — 提取书里没明说但能蒸馏出的新范畴
**Engine D · 书评聚合** — 真实他者声音，3 篇立场分散，评论也机理式深读

### 3. v0.3 Prompt 工具集（核心创新）

#### 要求式约束（强制结构，不限制语言）

**对每根力**：
- 必须含**反直觉案例**——把抽象判断落到具体场景里，最好能反驳常识
- 必须给**机理推导**——为什么必然如此，不是"这本书认为"
- 必须有**推到极端的延伸**——这条力推到底会怎样、它的边界在哪

**对整篇**：
- 必须提取**至少一个书里没明说但能从机理蒸馏出的概念**（不是把书里现成术语包装一下）
- 必须**勾连至少一对不同范畴**（如把"螺旋"和"物化"两个范畴连起来）
- 必须保留思想的**关键修饰条件**（如马克思主义认识论的"具体的、历史的、阶级的"——不能为简洁省成"具体的"）

**对评论**：
- 每条评论必须给：**评论者背景** + **这条评论戳的是哪个穴位** + **这条评论本身的局限或可被反驳的地方**
- 不只是摘原话+短评

#### 自检式（写完每段回头问）

- 这段如果给一个完全没读过这本书的人看，他能否从我的话里推导出书的机理？
- 如果换一本完全不同的书，这段会显得生硬吗？（如果会，说明套了模板，要重写）
- 我用了的术语，是用术语解开了机理，还是用术语替代了机理？
- 我提到的"反直觉案例"，真的反直觉吗？还是只是个普通例子？

### 4. 输出格式

```
# {书名} —— {一句话灵魂之问}

> {作者 · 出版年 · 一句话来源说明}

## 暗门
{含历史语境的具体细节，不只是抽象的"作者想回答 X"。要把作者写作时的真实处境讲清楚。}

## 三根力

### 力一：{用动态语言命名，不用术语}
{每根力 400-600 字。含：反直觉案例 + 机理推导 + 推到极端的延伸}

### 力二：...
### 力三：...

## 新增范畴
{每条 80-120 字。含：定义 + 日常用法。这些必须是从机理里蒸馏出来的，不是书里现成术语。}

## 他者之眼
**[推崇 · {作者}（{出处+背景}）]**
> {原话 1-2 句}
{200 字：评论者背景 + 这条评论戳的穴位 + 评论本身的局限或反思}

**[建设性批评 · ...]**
**[否定/反驳 · ...]**

## 一句话带走
{30-50 字的金句}
```

字数：目标 2500，深度优先，无硬限制。

## v0.3 验收标准

1. 用《实践论》在**不看用户示范**的前提下、纯靠 v0.3 prompt 工具集重新生成一份，对比深度是否独立扛得住
2. 用一本完全不同类型的书（如小说、工具书）再跑一份，验证 prompt 跨书鲁棒性
3. 评论部分能否独立达到机理深度（不靠拆书部分撑）

通过后 → GitHub repo 落地 + openclaw 自动接入（v0.4）。
