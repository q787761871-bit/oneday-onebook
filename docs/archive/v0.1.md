# 一天一本书 v0.1

## 动机

人无法感知到认知以外的事情。每天补一本书的「思想 + 方法论」，扩展大脑「范畴仓库」的量级。

对标 bestblog：openclaw 每日定时推送一份。

## 核心设计

### 1. 书目来源（c 方案）

目标：每天一本"够分量、够新鲜、不重复"的书。

候选信源（待选一确定）：
- **豆瓣读书** —— 每周新书速递 / 各分类近期热榜 / 9.0 高分新书
- **Goodreads** —— Choice Awards / Most Read This Week
- **微信读书** —— 神作榜 / 飙升榜
- **The New York Times Books** —— Editor's Choice
- **Marginalia / Brain Pickings** —— 选书品位高，但更新不快

> v0.1 建议：先锁 1 个主信源 + 1 个备选。**豆瓣是中文阅读的最优解**，覆盖度和品位都在。

筛选规则（每日选 1 本）：
- 排除已推过的书（去重 → 见 §4）
- 排除阿刚已经读过/熟悉的范畴（待建"已知范畴"清单）
- 优先选**思想密度高**的（哲学/认知/方法论 > 小说/传记/工具书）
- 同一作者一周内不重复

### 2. 解读双引擎

一份解读 = 拆书 + 降秩 两 prompt 串联。

**Engine A · 拆书（李继刚）**
> 透过现象看本质，找到作者夜不能寐的那个问题。
- 核心追求：找出作者真正回答的那个终极问题
- 输出：作者的"暗门"、灵魂之问、问题转向
- 在解读里的位置：**前 1/3，回答"这本书在回答什么"**

**Engine B · 降秩（ljg-rank）**
> 把领域降到几根独立的、不可再分的力。
- 核心追求：找出撑住整本书的几根生成器
- 输出：3-5 根力 + 它们如何组合生成书中的具体观点
- 在解读里的位置：**中 1/3，回答"它靠什么撑着"**

**Engine C · 范畴提取（自定义）**
> 提炼这本书带给"范畴仓库"的新增条目。
- 输出：3-5 个新范畴（带定义 + 一句话用法）
- 在解读里的位置：**末 1/3，回答"我大脑里多了什么"**

### 3. 输出格式（800 字）

```
# {书名} —— {一句话灵魂之问}

> 作者 · 出版年 · {一句话来源说明（豆瓣 9.x / NYT 选书）}

## 暗门
{50-100 字：这本书的"灵魂之问"是什么，作者在跟谁对话}

## 三根力
1. {力1} —— {一句话}
2. {力2} —— {一句话}
3. {力3} —— {一句话}
{每根力下面 50-80 字展开}

## 新增范畴
- **{范畴1}**：{定义 + 用法举例}
- **{范畴2}**：...
- **{范畴3}**：...

## 一句话带走
{30 字内的金句式总结}
```

总字数控制在 750-850 字。早餐时间能读完。

### 4. 沉淀与去重

**沉淀位置**：`02 你的阅读/笔记/_dailybook/YYYY-MM-DD_{书名}.md`

**去重机制**：
- 维护 `01 你的项目/一天一本书/_推过的书.md`（书名 + ISBN + 推送日）
- 每次选书前先 grep 这份清单
- 清单按月分块，避免无限增长

### 5. 推送通道

**复用 openclaw bestblog 通道**（待调研现有实现细节）：
- openclaw LaunchAgent 已常驻（端口 18789，deepseek-v4-pro）
- bestblog 现有推送时间/格式/送达方式 → 套用
- 「一天一本书」可视为 bestblog 的一个新 channel

待调研项：
- bestblog 现在通过什么方式推送（飞书 bot / 邮件 / 其他）
- 推送频率和时间
- 是否已有"多 channel"的抽象，能否平滑加入

## v0.1 验收标准

1. 手工跑通首本（不接 openclaw，先验 prompt 组合的产出质量是否够"扩展范畴"的标准）
2. 800 字格式稳定，不漂
3. 推过的书写入去重清单
4. 沉淀文件命名/格式定型

接通 openclaw 自动推送 → 留给 v0.2。
