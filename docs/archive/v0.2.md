# 一天一本书 v0.2

## 与 v0.1 的差异

| 项 | v0.1 | v0.2 |
|----|------|------|
| 解读引擎 | 拆书 + 降秩 + 范畴提取（3 个） | + **书评聚合**（4 个） |
| 字数 | 800 | 1200 |
| 视角来源 | 全是 AI 生成 | 引入真实他者声音 |
| 实施方式 | 纯 LLM | LLM + agent-reach 信源拉取 |

## 动机回顾

人无法感知到认知以外的事情。每天补一本书的「思想 + 方法论」，扩展大脑「范畴仓库」的量级。

v0.2 增量动机：**光听作者自己说话，会被作者的"暗门"带跑**。引入真实读者/评论家的声音，才能看见作者看不见的地方。

## 核心设计

### 1. 书目来源

不分流策略 —— 经典书和新书都走同一管道，命中即用。

主信源（按优先级）：
- **豆瓣读书** —— 中文阅读最优解，每周新书速递 / 9.0 高分新书 / 各分类近期热榜
- 备选：NYT Books、Goodreads、Marginalia

筛选规则（每日选 1 本）：
- 排除已推过的书（去重 → 见 §5）
- 排除阿刚已熟悉的范畴（v0.1 暂跳过，攒几本后反推清单）
- 优先**思想密度高**的（哲学/认知/方法论 > 小说/传记/工具书）
- 同一作者一周内不重复

### 2. 解读四引擎

一份解读 = 4 个引擎串联。

**Engine A · 拆书（李继刚 jigang:references:拆书）**
> 透过现象看本质，找到作者夜不能寐的那个问题。
- 输出：作者的"暗门"、灵魂之问、问题转向
- 在解读里的位置：**前段，回答"这本书在回答什么"**

**Engine B · 降秩（ljg-rank）**
> 把领域降到几根独立的、不可再分的力。
- 输出：3-5 根力 + 它们如何组合生成书中的具体观点
- 在解读里的位置：**中段，回答"它靠什么撑着"**

**Engine C · 范畴提取（自定义）**
- 输出：3-5 个新范畴（带定义 + 一句话用法）
- 在解读里的位置：**后段，回答"我大脑里多了什么"**

**Engine D · 书评聚合（v0.2 新增）**
> 引入真实他者声音，不 AI 模拟、不重写。
- 拉取 3 篇真实书评（立场分散：推崇 / 中立 / 批评 至少各 1 篇）
- 每篇做：出处标注 + 核心观点摘要（1-2 句直引或转述）+ 与本书"暗门"的对话关系
- 在解读里的位置：**末段，回答"别人怎么看"**

### 3. 书评信源与拉取

**统一管道，并行尝试**：

| 信源 | 优先级 | 适合 | 工具 |
|------|--------|------|------|
| 维基百科 Reception 节 | 高 | 经典书神器 | agent-reach (web) |
| 豆瓣读书长评 | 高 | 中文新书/经典通吃 | agent-reach (web/social) |
| NYT Books / NYRB / LRB | 中 | 英文版书 | agent-reach (web) |
| 学术期刊（JSTOR/CNKI） | 低（v0.2 暂不接） | 经典/学术专著 | 手动 |

**筛选规则（豆瓣长评的"非水"信号）**：
- 字数 ≥ 1000
- 赞数 ≥ 100（或赞:评 比例正常）
- 写在豆列/专栏里的优先
- 排除明显的"剧透+夸赞"模板

**3 篇立场分布**：
- 至少 1 篇推崇（理解作者的暗门并扩展之）
- 至少 1 篇中立或建设性批评（指出作者方法的边界）
- 至少 1 篇否定/反驳（指出作者的盲点或时代局限）

如果某类立场拉不到，AI 不模拟，**如实标注"未拉到 XX 立场书评"**。

### 4. 输出格式（1200 字）

```
# {书名} —— {一句话灵魂之问}

> 作者 · 出版年 · {一句话来源说明}

## 暗门
{150 字：这本书的"灵魂之问"是什么，作者在跟谁对话}

## 三根力
1. {力1} —— {一句话}
2. {力2} —— {一句话}
3. {力3} —— {一句话}
{每根力下面 80-120 字展开}

## 新增范畴
- **{范畴1}**：{定义 + 用法举例}
- **{范畴2}**：...
- **{范畴3}**：...
- **{范畴4}**：...

## 他者之眼
**[推崇 · {作者}（{出处}）]**
> {原文摘录或精确转述,1-2 句}
{30-50 字：这篇评论与本书"暗门"的对话关系}

**[中立/建设性批评 · {作者}（{出处}）]**
> {同上}
{同上}

**[反驳/否定 · {作者}（{出处}）]**
> {同上}
{同上}

## 一句话带走
{30 字内的金句式总结}
```

总字数控制在 1100-1300 字。早餐+午休能消化完。

## v0.2 验收标准

1. 用《实践论》试跑首份带书评的 v0.2 解读，1200 字格式稳定
2. 书评 3 篇立场分布合理（不全是好评）
3. agent-reach 拉书评的可行性确认（豆瓣 + 维基 至少各能拉一种）
4. v0.2 沉淀文件命名/格式定型

通过后 → 进入 GitHub repo 建设 + openclaw 自动接入（v0.3）。
