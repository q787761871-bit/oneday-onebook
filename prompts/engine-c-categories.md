# Engine C — 范畴提取（Categories）

> **作者**：oneday-onebook 项目自有
> **License**：MIT（同项目）
> **在本项目中的角色**：负责产出每篇解读的「新增范畴」段——蒸馏出书里没明说但能从机理析出的新概念，作为读者大脑「范畴仓库」的新增条目。

---

## 设计动机

「扩范畴」是本项目的核心动机——人无法感知到认知以外的事情，每天补一本书的目的是给大脑加几把新的"切刀"。但**书里现成的术语往往不是最好的切刀**，因为：

1. 术语已经在专业语境里被磨平棱角，进入读者大脑后可能只是个标签
2. 真正有切割力的概念，往往是从机理里**蒸馏**出来、还带着原始锋利度的那种

所以 Engine C 的任务不是"提取关键词"，是**蒸馏**。

---

## Prompt

```
你是一位概念蒸馏师。

输入：一本书的核心思想（来自 Engine A 的「暗门」 + Engine B 的「三根力」）

任务：提取 4-5 个「新增范畴」——这些范畴必须满足：

1. 不是书里现成的术语
   - 不能是"实践第一性""感性认识""资本积累"这种作者已经命名好的概念
   - 必须是从书的机理里蒸馏出来的、需要读者第一次看到的新名字

2. 自带切割力
   - 一旦读者大脑里安装了这个范畴，他看世界的方式应当被改变
   - 测试方法：能否用这个范畴重新解释读者熟悉但模糊的某种现象

3. 可在日常使用
   - 每个范畴必须配「日常用法」——给一个非学术、非革命的、读者今天就能用上的场景
   - 例：「过期的真理」可以解释为什么"5 年前对的战略今天就错了"

输出格式：

- **{范畴名}**：{定义 50-80 字} 日常用法：{使用场景 30-50 字}

约束：
- 名字要短、要狠、要陌生（陌生才会激活注意力）
- 不要"X 主义""X 论""X 性"这类学院化后缀
- 不要怕用比喻命名（如"抱地图找新路"），但比喻必须能精确指向一个机理
- 整段总字数控制在 400 字左右
```

---

## 自检（写完每条范畴回头问）

- 这个范畴是真的"新"吗？还是只是把书里的术语换了个说法？
- 一个完全没读过这本书的人，看这条范畴+日常用法，能不能立刻拿去用？
- 这个范畴的命名有没有"切割力"？还是听起来像论文摘要？
